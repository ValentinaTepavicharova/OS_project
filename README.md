
##  1. Структура на проекта
Проектът е организиран в следната файлова структура, която разделя сорс кода на уеб приложението от конфигурационните файлове за контейнеризация:

* `img/` — Директория, съдържаща изображенията на продуктите за онлайн магазина.
* `index.php` — Начална страница на приложението, която показва каталога с продукти.
* `cart.php` — Страница за преглед на пазарската количка и калкулиране на общата сума.
* `add.php` & `remove.php` — Скриптове, управляващи добавянето и премахването на артикули в сесията.
* `db.php` — Конфигурационен файл за установяване на PDO връзка с базата данни.
* `my_shop.sql` — SQL скрипт с първоначалната структура на базата данни и примерни данни.
* `Dockerfile` — Инструкции за изграждане на персонализиран Docker образ за PHP приложението.
* `compose.yml` — Файл за дефиниране и управление на многоконтейнерната архитектура.

---

##  2. Как работи всеки от компонентите

1. **Уеб приложение (PHP + Apache):**
   * **Потребителски интерфейс:** Изграден с Bootstrap 5, осигуряващ визуализация на продуктите под формата на карти.
   * **Управление на количката:** Използват се native PHP сесии (`$_SESSION`), което позволява съхранение на състоянието за избраните от потребителя продукти без нужда от временни записи в базата данни.

2. **База данни (MySQL 8.0):**
   * Използва официален релационен образ на MySQL. Тя съхранява таблицата `products` (с полета за `id`, `name`, `price` и `image`) и обслужва SQL заявките, изпратени от PHP приложението за извличане на актуалния каталог.

##  3. Комуникация между услугите

Мрежовата комуникация е изолирана и автоматизирана чрез Docker Compose:
* **Име на хоста (DNS):** Във файла `db.php` за адрес на хоста е зададено директно името на услугата: `$host = 'db';`. Вграденият в Docker DNS сървър автоматично резолва това име до вътрешния IP адрес на MySQL контейнера.
* **Инициализация:** Чрез обвързване (Volume mount) на `my_shop.sql` към директорията `/docker-entrypoint-initdb.d/` в MySQL контейнера, базата данни автоматично изпълнява скрипта и се конфигурира при първоначално стартиране.
* **Зависимост:** Услугата за уеб сървъра изчаква стартирането на базата данни посредством директивата `depends_on`.
